学习总结

这次大语言模型智能体的学习与实践，不仅是一次技术能力的提升，更是一次对 “智能” 本质的探索 —— 智能体的价值不在于 “无所不能”，而在于 “在已知边界内，高效连接推理与行动，持续逼近目标”。而亲手构建的过程，正是理解这一本质、并不断拓展其可能空间的最佳路径。

习题

1.ReAct、Plan-and-Solve 和 Reflection

（1）本质区别：
ReAct 模式：边想边做的探索者
阐述 ReAct“边想边做，边做边学” 的核心思想，以找物品为例，说明其想法、行动、观察的循环过程，总结其灵活应变但缺乏长期目标把控的特点。
Plan-and-Solve 模式：谋定而后动的规划者
解释该模式先规划再执行的流程，以复杂任务为例，体现规划调整过程，得出其适合多步骤、强依赖任务，但应对突发情况灵活性欠佳的结论。
Reflection 模式：自我反思的进步者
介绍其在执行中不断评估过往行为优化后续决策的特点，以写作、解题等场景为例说明基础反思过程，以及带强化学习的反思机制如何打分和调整策略。
（2）设计智能家居助手
对比三种范式，ReAct虽能实时响应但缺乏整体规划。Reflection 主要用于优化结果而非直接控制，Plan-and-Solve 适合复杂任务规划与调整，更符合助手需求。
因为：如能对设备控制制定完整计划，执行中根据情况调整，可满足多设备协同和按用户习惯自动调节。
（3）构建以 Plan-and-Solve 为主体，融入 ReAct 处理突发情况，利用 Reflection 优化决策的架构，阐述各部分工作流程和交互方式。像平常家庭管理用Plan-and-Solve，家庭成员出现生命安全或者家内煤气超标，断电可以用ReAct，再利用Reflection优化下一步行动。

2.正则表达式来解析大语言模型的输出

（1）核心脆弱性
格式敏感：严格依赖 Thought:「Action: 等固定标识符，模型输出少空格、多标点或换行会直接失效。 语义鲁棒性差：仅匹配字符串形式，无法理解模型输出的语义逻辑，比如 Action 参数缺失却符合正则格式时会误解析。 扩展性不足：新增工具或修改输出结构时，需重新编写正则表达式，维护成本随场景复杂度上升。 典型失败场景 （2）更鲁棒的输出解析方案 模型输出格式变体：如 Thought - xxxx「Action：xxxx」（符号替换）、多段换行分隔 Thought 和 Action。 内容嵌套干扰：Action 参数中包含 :「} 等正则关键字符，导致匹配截断或全量失败。
逻辑不完整输出：模型仅生成 Thought 未生成 Action，或 Action 缺少必要参数，正则仍会提取残缺信息。
多轮交互累积误差：多轮对话中模型输出格式逐渐偏移，正则无法自适应调整。

4.Plan-and-Solve 范式将任务分解为”规划”和”执行”两个阶段。

（1）如何设计一个”动态重规划”机制？
反馈驱动：以执行结果（成功 / 失败、实际输出 vs 预期目标）作为重规划触发信号。
最小修改：仅调整受影响的步骤，不重构整个计划，平衡效率与稳定性。
目标守恒：重规划后需保证最终目标不变，避免偏离核心任务。
（2）Plan-and-Solve 与 ReAct:在处理”预订一次从北京到上海的商务旅行（包括机票、酒店、租车）”这样的任务时，哪种范式更合适？为什么？
Plan-and-Solve 范式更合适。核心原因是该任务的 “多步骤强依赖、目标明确、需全局统筹” 特性，与 Plan-and-Solve 的全局规划优势高度适配，而 ReAct 的 “边想边做” 模式易出现逻辑混乱或资源浪费。
（3）尝试设计一个”分层规划”系统:先生成高层次的抽象计划，然后针对每个高层步骤再生成详细的子计划。这种设计有什么优势？
提升规划效率：解释分层规划通过减少不必要细节，快速确定关键步骤，节省时间精力，提高规划速度。
增强灵活性与适应性：阐述面对变化时，分层规划只需调整特定层次计划，无需全盘修改，能更好应对突发情况。
提高规划质量：说明分层规划从宏观到微观思考，全面考虑各层次关系，使计划更合理、全面、科学。
便于团队协作与沟通：分析分层规划按层次分配任务，成员明确职责，各层次计划成为沟通协作依据，减少冲突误解。

5.Reflection 的”执行-反思-优化”循环机制。

（1）如果使用两个不同的模型会带来什么影响？
强大模型用于反思的优势 快速模型用于执行的优势 模型间的协作流程与交互机制
（2）Reflection 机制的终止条件是”反馈中包含无需改进”或”达到最大迭代次数”。这种设计是否合理？能否设计一个更智能的终止条件？
现有设计基本合理但不够智能，更优方案需结合 “改进价值 + 反馈质量 + 迭代效果” 多维度判断。以 “改进价值是否为正” 为核心，结合反馈质量、迭代趋势、资源约束，形成复合终止规则。

两者结构差异精准匹配各自 “动态交互” 与 “先规划后执行” 的核心逻辑，直接决定任务推进方式。
（2）Reflection 提示词中，我们使用了”你是一位极其严格的代码评审专家”这样的角色设定。尝试修改这个角色设定（如改为”你是一位注重代码可读性的开源项目维护者”），观察输出结果的变化，并总结角色设定对智能体行为的影响。
角色设定为智能体提供了 “行为锚点”，明确了任务中的优先级排序。
角色的专业背景决定了评判维度，比如 “评审专家” 侧重 “纠错”，“维护者” 侧重 “协作”。
角色的语气特质会传递给智能体，影响输出的沟通风格

习题

1

一、马尔可夫假设的含义
马尔可夫假设是 N-gram 模型的基础前提，核心含义是：一个词的出现概率仅取决于它前面紧邻的 N-1 个词，与更早的词无关。
比如 2-gram（二元模型）假设当前词概率只依赖前 1 个词，3-gram（三元模型）依赖前 2 个词。
本质是通过 “局部依赖简化”，将复杂的全局语言依赖关系转化为可计算的局部窗口依赖，降低建模难度。
二、N-gram 模型的根本性局限
上下文窗口固定，无法捕捉长距离依赖：N 值一旦确定，只能关注紧邻的 N-1 个词，无法处理跨多个词的语义关联（如长句中前后指代、逻辑呼应）。
数据稀疏问题严重：N 越大，需要的训练数据量呈指数级增长，大量 N-gram 组合在语料中从未出现，导致概率估计不准确。
泛化能力差：仅依赖词的表面共现，无法捕捉词的语义相似性（如 “手机” 和 “电话” 语义相近，但 N-gram 会视为完全不同的上下文）。
参数规模庞大：N 增大时，模型参数数量随词汇表大小呈指数增长，存储和计算成本极高。
三、RNN/LSTM 如何克服 N-gram 局限
核心思路：建模序列动态依赖
不再局限于固定 N-1 窗口，而是通过 “隐藏状态” 累计前文信息，理论上可捕捉任意长度的上下文依赖。
隐藏状态会随序列推进不断更新，将历史信息传递到当前步，替代 N-gram 的固定窗口。
各自优势
RNN（循环神经网络）
优势：首次实现序列数据的 “记忆传递”，突破 N-gram 的固定窗口限制，参数规模远小于 N-gram（与词汇表大小线性相关）。
适用于短序列场景，结构简单、易于理解和实现。
LSTM（长短期记忆网络）
优势：通过门控机制（输入门、遗忘门、输出门）解决 RNN 的 “梯度消失 / 爆炸” 问题，能有效捕捉长序列中的关键依赖（如长句前后语义关联）。
相比 RNN，对长文本的建模稳定性更强，是早期处理长序列语言任务的核心模型。

2

  1. 自注意力机制：让每个 token 关注全局信息，动态分配权重，核心是 “全局依赖 + 并行计算”；
  2. Transformer 并行性：自注意力无顺序依赖，RNN 有前序依赖；位置编码解决自注意力的 “顺序失忆” 问题；
  3. Decoder-Only 架构：适合自回归生成，数据简单、结构简洁、多任务能力强，成为大模型主流；

3

子词分词（BPE）：平衡词汇表大小和语义表达，解决字符级的低效和单词级的 OOV 问题，是大模型输入处理的核心。

4

若预算有限、隐私要求高、需深度定制，优先选择 开源模型本地部署；
• 若追求高性能、低运维成本、快速上线，可选择 闭源模型 API；
• 最实用的方案是 混合架构：用开源模型覆盖常规场景，闭源模型处理复杂任务，既控制成本和隐私，又保证用户体验。

5

缓解大语言模型幻觉是一个系统性工程，单一方法难以完全解决问题。实际应用中，建议结合多种策略（如 RAG + 工具调用 + 幻觉检测），根据具体任务场景选择最合适的组合。未来，随着研究的深入，模型的事实准确性将不断提升，为更可靠的 AI 应用奠定基础。

6

设计一个优秀的论文辅助阅读智能体是一个系统工程，需要在模型选择、提示词工程、长文本处理和事实性保障等多个维度进行精心设计。通过采用强大的基座模型、结构化的提示策略、分块融合的长文本处理流程以及多重事实校验机制

学习经验总结

核心收获：1）智能体的本质是 “感知 - 思考 - 行动 - 观察” 闭环，核心是 “自主性”；2）从传统智能体到 LLM 驱动新范式，按决策架构、时间反应性、知识表示可精准分类；3）LLM 智能体实现核心是 “提示工程 + 工具调用 + 循环迭代”，PEAS 模型可清晰定义任务环境。

习题

智能体判断与类型分析

• case A：不属于，仅具算力，无感知、行动能力和目标。
• case B：属于，反应式 + 基于模型的反射智能体，传感器感知障碍物，执行器快速避障，目标是安全行驶。
• case C：属于，规划式 + 学习型智能体，规划多步棋路，通过强化学习进化，目标是赢棋。
• case D：属于，混合式 + 神经符号主义智能体，即时回应 + 工具查询，动态解决投诉，目标是安抚用户。

智能健身教练的 PEAS 模型与环境特性

• PEAS 模型：性能度量（目标达成率、安全性、满意度）；环境（用户身体状态、运动场景、设备反馈）；执行器（语音指导、计划生成、饮食建议）；传感器（可穿戴设备、用户交互接口）。
• 环境特性：部分可观察、随机性、动态性、序贯性。

售后退款方案对比与混合方案

• 方案 A（Workflow）：优点（高效、可解释、成本低）；缺点（刚性强、适配差），适合标准化场景。
• 方案 B（Agent）：优点（灵活、个性化、自学习）；缺点（开发难、黑箱、成本高），适合复杂场景。
• 方案 C（混合）：常规场景用 Workflow，例外 / 复杂场景用 Agent，人工兜底 + 反馈优化。

旅行助手功能扩展

• 记忆功能：新增偏好存储模块，将偏好嵌入 TAO 循环的 prompt。
• 售罄备选：工具新增门票查询，识别售罄后重新调用工具推荐同类型景点。
• 3 次拒绝调整：记录拒绝次数，触发反思，询问偏好或切换推荐类型。

Understanding of Markdown

第一次接触markdown

第一次来到这里，听到学长说的还有教学中的md，我是茫然的，这是什么东西啊？？
当我去了解了一下，我逐渐开始明白这个东西有多么好用

对md的理解

让作者专注文字而非排版，通过轻量级符号实现高效格式化文档。

方便性

1. 学习成本低：语法简单易记
2. 编写效率高：键盘不离开主键区
3. 跨平台兼容：纯文本通用性
4. 转换灵活：支持导出多种格式
5. 版本控制友好：适合Git管理

对新手十分友好

最打动新手的应该是：五分钟就能完成第一篇格式化文档。这需要拆解三个层面：一是语法记忆量极小（10个符号覆盖90%场景），二是即时反馈的成就感（输入符号后实时预览效果），三是容错率高（漏空格也能被纠正）。
